🛰️   AI-Powered Road Monitoring

Automated Pothole
Detection System

A computer-vision system that identifies road damage in real-time and calculates the exact construction materials needed for repair.

What this project does
🔍

Real-Time Detection

Analyses images from road cameras using a YOLOv8 INT8 TFLite model trained specifically on pothole imagery.

📐

Area Estimation

Calculates physical pothole area in m² using pixel-to-metre calibration based on known road width.

🏗️

Material Calculation

Instantly computes cement, sand, and aggregate quantities needed for repair — eliminating manual estimation.

🗺️

GPS Mapping SOON

Future GPS integration will pin every detection on a live map, enabling city-wide road health monitoring.

How it works
01

Image Capture & Upload

A camera (dashcam, drone, fixed CCTV) sends images to the server via the /api/detect REST endpoint.

02

YOLOv8 Inference

The image is resized to 160×160, normalised, and passed through the INT8 quantized YOLOv8 model. Output is 525 anchor boxes with confidence scores.

03

NMS Filtering

Non-Maximum Suppression removes overlapping boxes. Only detections above the confidence threshold (0.45) are kept.

04

Repair Estimation

Bounding box pixels → m² via road-width calibration. Volume at 8 cm depth → cement (350 kg/m³), sand (700 kg/m³), aggregate (1050 kg/m³).

05

Logging & Dashboard

Results stored in SQLite. Admin dashboard shows annotated images, material totals, and detection history.
